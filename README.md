# SUPERIOR CRM

Минимальная CRM-система для студии персонального тренинга SUPERIOR.

Этот репозиторий содержит упрощённую реализацию расписания и записи клиентов.

## Текущее состояние реализации

- Backend: FastAPI
- ORM: SQLAlchemy
- База данных: SQLite (файл `superior.db` в корне проекта)
- Шаблоны: Jinja2
- Запуск: Uvicorn

В настоящий момент реализованы базовые модели `Client`, `Slot`, `Booking` и три маршрута:

- `GET /` — список слотов (шаблон `templates/schedule.html`)
- `GET /slot/{slot_id}` — страница слота и список записанных (шаблон `templates/slot.html`)
- `POST /slot/{slot_id}/add` — форма добавления клиента в слот

Примечание: в навигации шаблона есть ссылка `/clients`, но соответствующий эндпоинт пока не реализован.

## Быстрый запуск (локально)

1. Создайте виртуальное окружение и активируйте его:

```powershell
python -m venv venv
venv\Scripts\activate
```

2. Установите зависимости:

```powershell
pip install -r requirements.txt
```

3. Запустите приложение в режиме разработки:

```powershell
uvicorn app:app --reload
```

4. Откройте в браузере: http://127.0.0.1:8000/

## База данных

При первом запуске скрипт создаёт файл `superior.db` и добавляет тестовые записи (несколько клиентов и слотов).
Чтобы сбросить данные — остановите сервер и удалите файл `superior.db`.

## Ограничения и замечания

- Используется SQLite для простоты локальной разработки; для продакшена рекомендуется перейти на PostgreSQL и настроить миграции (Alembic).
- Сессии SQLAlchemy создаются прямо в обработчиках — стоит вынести получение сессии в зависимость FastAPI и корректно закрывать её.
- Нужна валидация входных данных (Pydantic-схемы) и обработка ошибок (404, 400).
- В текущей реализации возможны гонки при одновременных бронированиях; для высоконагруженных систем требуется транзакционная защита и/или ограничения на уровне БД.
